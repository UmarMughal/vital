Navigation Header

The navigation bar is fully responsive and can host an unlimited amount of menu items.

Sub-menu items are hidden by default on $tablet and smaller, but you can disable this by editing or removing line 131 in _header.sass.

.row.header
  .section
    nav
      = link_to 'index.html', class: 'logo' do
        i.icon-vital.i3x
      label#menu-toggle-label for='menu-toggle'
      input#menu-toggle type='checkbox'
      ul.menu
        li = link_to 'Link', '#'
        li
          = link_to 'With Menu', '#'
          ul
            li = link_to 'Menu Item', '#'
            li = link_to 'Menu Item', '#'

Forms

Cross-platform onsistency is a very important factor when Vital was concepted, so we created form elements that will look and feel consistent on most platforms while retaining the ability to gracefully degrade.

Form fields that are nested under the form tag are automatically full in width. Optionally you can use the class .full-width if you do not use a form tag.

We developed custom radios and checkboxes because standard checkboxes didn't scale well. As a side benefit, you can customize the look and feel to be more in line with your application's branding.

Vital Iconograpgy

Vital comes with a few of it's own set of font icons. Some components such as checkboxes and the alert bar are dependant on them.

Most font icon libraries simply have either too many icons that result in very large files. We've included a Font Custom template for you to be able to quickly generate your own unique set of iconograpgy from SVG files.

For more information on creating and compiling vectors, visit Font Custom

Please note that some icons (such as those with social names) are blocked by ad-blockers.
